# Виправлення помилки доступу до галереї

## Проблема
При натисканні "Вибрати з галереї" виникає помилка:
```
IOSurfaceClientSetSurfaceNotify failed e00002c7
```

Галерея не відкривається.

## Причина
iOS вимагає явного опису використання фото в файлі `Info.plist` для доступу до бібліотеки фото користувача.

## Рішення

### Варіант 1: Через Xcode Interface

1. Відкрийте файл `Info.plist` у Xcode
2. Натисніть правою кнопкою миші → **Add Row**
3. Додайте наступний ключ:
   - **Key**: `Privacy - Photo Library Usage Description`
   - **Type**: String
   - **Value**: `Ми потребуємо доступ до вашої галереї, щоб ви могли вибрати фото профілю`

### Варіант 2: Через Source Code

Якщо ви редагуєте `Info.plist` як XML, додайте:

```xml
<key>NSPhotoLibraryUsageDescription</key>
<string>Ми потребуємо доступ до вашої галереї, щоб ви могли вибрати фото профілю</string>
```

### Варіант 3: Через Target Settings (рекомендовано для нових проектів)

1. Виберіть ваш проект в Xcode
2. Виберіть Target → CareerKids
3. Перейдіть на вкладку **Info**
4. В секції **Custom iOS Target Properties** натисніть **+**
5. Виберіть зі списку: `Privacy - Photo Library Usage Description`
6. Встановіть значення: `Ми потребуємо доступ до вашої галереї, щоб ви могли вибрати фото профілю`

## Повний список необхідних ключів

Для повної підтримки вибору фото додайте ці ключі:

```xml
<!-- Доступ до бібліотеки фото (обов'язково) -->
<key>NSPhotoLibraryUsageDescription</key>
<string>Ми потребуємо доступ до вашої галереї, щоб ви могли вибрати фото профілю</string>

<!-- Доступ до додавання фото (опціонально, якщо плануєте дозволити зберігати фото) -->
<key>NSPhotoLibraryAddUsageDescription</key>
<string>Дозвольте додатку зберігати фото у вашій галереї</string>

<!-- Доступ до камери (якщо плануєте додати можливість робити фото) -->
<key>NSCameraUsageDescription</key>
<string>Ми потребуємо доступ до камери, щоб ви могли зробити фото профілю</string>
```

## Перевірка після додавання

1. **Видаліть додаток** з симулятора/пристрою
2. **Пересоберіть** проект (Product → Clean Build Folder)
3. **Запустіть** додаток знову
4. При першому натисканні "Вибрати з галереї" iOS покаже системний діалог з запитом дозволу
5. Дозвольте доступ
6. Галерея повинна відкритися успішно

## Альтернативне рішення (якщо Info.plist відсутній)

У нових проектах Xcode 14+ інколи Info.plist може бути не створений явно. В цьому випадку:

1. File → New → File
2. Виберіть **Property List**
3. Назвіть його `Info.plist`
4. В Target Settings → Build Settings знайдіть `Info.plist File`
5. Встановіть шлях до вашого Info.plist (наприклад, `CareerKids/Info.plist`)

## Важливо! 🔴

Без цих налаштувань:
- ❌ Галерея не відкриється
- ❌ Додаток може крашнутися
- ❌ В консолі будуть помилки про відсутність permissions

З цими налаштуваннями:
- ✅ iOS покаже користувачу зрозуміле повідомлення
- ✅ Користувач зможе дозволити/заборонити доступ
- ✅ PhotosPicker працюватиме коректно
